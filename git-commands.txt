git --version , git status 
git status then git init (iniitalization of the repo)
git add <file1> <file2>
git add .    (for all files to be added)
git commit
git log   (only tells about the history of current branch)
git log --all (tells about all branches history)
git log --oneline
git restore --staged <filename> (to remove the staged file from staging)
git config --global core.editor "notepad"   (this will open notepad instead of vim (that is the git commit editor))
git ignore and git keep 
git branch (tells at which branch)
git branch <branch_name>  (make new branch)
git switch <branch_name>   (takes the head to the branch_name) (you dont see stuff of other branches in your current branch damn... )
git branch -m <old-branch-name> <new-branch-name>
git branch -d <branch-name>
git diff --staged
git diff <branch-name-one> <branch-name-two>
git diff branch-name-one..branch-name-two
git diff <commit-hash-one> <commit-hash-two>
git stash   (gives temporary memory so you can switch between branches without commiting and then later commit the changes)
git stash save "work in progress on X feature" (saving with name)
git stash list  (all the stash currently)
git stash apply (apply the top stash)
git stash apply stash@{0} (apply the stash according to your choice)
git stash pop  (it applies and then removes the stash at once)
git stash drop
git stash apply stash@{0} <branch-name>  (applies stash to specific branch)
git stash clear
git tag <tag-name>
git tag -a <tag-name> -m "Release 1.0"  (annotated tags)
git tag  (list of all tags)
git tag <tag-name> <commit-hash> (tagginh specific hash code)
git tag -d <tag-name> (deleting tags)
